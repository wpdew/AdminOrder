# Prompts Index

## Описание

Эта папка содержит коллекцию промптов для работы с проектом AdminOrder Dashboard. Каждый промпт описывает конкретный аспект архитектуры или паттерн разработки.

## Список промптов

### 01-architecture.md
**Назначение**: Архитектура и структура проекта

Содержит:
- Принципы MVC паттерна
- Структура namespace
- Layout система  
- Ключевые файлы проекта
- Правила разработки
- Примеры добавления нового функционала

**Когда использовать**: При создании новых страниц, контроллеров, моделей

---

### 02-database.md
**Назначение**: Работа с базой данных SQLite

Содержит:
- Получение подключения через Singleton
- Выполнение запросов (SELECT, INSERT, UPDATE, DELETE)
- Структура моделей
- Миграции
- Примеры типовых запросов (JOIN, GROUP BY, транзакции)
- SQLite команды для CLI
- Правила безопасности
- Решение типовых ошибок

**Когда использовать**: При работе с данными, создании моделей, миграциях

---

### 03-mvc-patterns.md
**Назначение**: MVC паттерны и layout система

Содержит:
- Структура MVC
- BaseController с методом render()
- Layout система (main.php, auth.php)
- Примеры контроллеров
- Примеры views
- Точки входа
- Передача данных в view
- Flash сообщения
- Лучшие практики

**Когда использовать**: При создании страниц, работе с шаблонами, рефакторинге

---

### 04-auth-security.md
**Назначение**: Авторизация и безопасность

Содержит:
- Система Auth (login, logout, check)
- Защита страниц админки
- Процесс авторизации и регистрации
- Безопасность паролей (hashing, verification)
- XSS защита
- SQL Injection защита
- CSRF защита
- Session безопасность
- Валидация данных
- Rate Limiting
- Логирование безопасности
- Checklist безопасности

**Когда использовать**: При работе с авторизацией, формами, пользовательским вводом

---

## Как использовать промпты

### Для AI агента:
```
Прочитай промпт 03-mvc-patterns.md и создай страницу Products по этому паттерну
```

### Для разработчика:
Открой нужный промпт и следуй инструкциям для решения конкретной задачи.

### Для новичка в проекте:
Начни с чтения в порядке:
1. **01-architecture.md** - общая картина
2. **03-mvc-patterns.md** - как писать код
3. **02-database.md** - работа с данными
4. **04-auth-security.md** - безопасность

## Добавление новых промптов

При расширении проекта создавай новые промпты для документирования паттернов:

**Формат названия**: `{номер}-{краткое-название}.md`

**Структура промпта**:
```markdown
# Промпт: {Название}

## Описание
Краткое описание для чего этот промпт

## Основные концепции
Теория и принципы

## Примеры кода
Рабочие примеры

## Лучшие практики
Что делать и чего избегать

## Когда использовать
Сценарии применения
```
